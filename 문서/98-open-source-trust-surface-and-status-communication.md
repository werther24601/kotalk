# Open Source Trust Surface And Status Communication

## 목적

오픈소스 프로젝트의 신뢰는 기능보다 상태 공개 방식에서도 만들어진다. 이 문서는 공개 저장소 표면의 신뢰 커뮤니케이션 기준을 정의한다.

## 핵심 요소

- 실제 상태만 적는 README
- 최신 스크린샷 유지
- Known Gaps 공개
- 릴리즈별 안정성 요약

## 원칙

- 과장된 약속보다 현재 한계를 먼저 적는다.
- 라이브, Windows, Android 상태를 따로 쓴다.
- 사용자 관점 리뷰를 문서 안에 남긴다.

## 성공 기준

- 저장소를 읽은 사람이 제품 상태를 과대평가하지 않는다.
