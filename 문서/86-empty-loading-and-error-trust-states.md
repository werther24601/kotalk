# Empty, Loading, And Error Trust States

## 목적

사용자가 서비스를 믿는지는 정상 상태보다 막힘 상태에서 더 빨리 드러난다. 이 문서는 빈 상태, 로딩, 오류 상태를 신뢰 중심으로 정의한다.

## 원칙

- 빈 상태와 오류 상태는 명확히 다르게 보여 준다.
- 로딩은 기다림보다 `무엇을 준비 중인지`를 알려야 한다.
- 오류는 원인보다 다음 행동을 먼저 안내한다.

## 상태별 기준

- 빈 상태: 첫 행동 CTA 제공
- 로딩: 데이터 종류와 예상 결과를 짧게 노출
- 복구 가능 오류: 재시도와 현재 화면 유지
- 복구 불가 오류: 왜 다시 들어와야 하는지 설명

## 성공 기준

- 사용자는 멈춘 것과 비어 있는 것을 혼동하지 않는다.
