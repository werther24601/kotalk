# Open Source Community Issue Triage Guidelines

## 목적

오픈소스 프로젝트답게 이슈와 제안, 버그 리포트를 체계적으로 다루기 위한 기준을 정의한다.

## 분류

- bug
- ux
- design
- docs
- release
- security
- good first issue

## 우선순위

- P0: 로그인/전송/세션/다운로드 치명 문제
- P1: 반복 사용을 막는 UX 문제
- P2: 품질 향상과 구조 개선
- P3: 아이디어/후속 개선

## 처리 원칙

- 재현 가능성 기록
- 사용자 영향 설명
- 실제 상태 문서 반영 여부 확인

## 완료 기준

- 이슈 목록이 혼란스러운 아이디어 저장소가 아니라 제품 개선 보드처럼 작동해야 한다.
