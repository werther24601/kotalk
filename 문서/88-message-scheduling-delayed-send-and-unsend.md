# Message Scheduling, Delayed Send, And Unsend

## 목적

업무용과 일상용 모두에서 `지금은 쓰되 나중에 보내기`가 필요하다. 이 문서는 예약 전송과 지연 전송의 최소 규칙을 정의한다.

## 기능 범위

- 10초 지연 전송
- 시간 지정 예약 전송
- 보내기 취소

## 원칙

- 취소 가능 시간은 명확해야 한다.
- 예약 기능은 작성 흐름을 복잡하게 만들지 않는다.
- 업무 대화에서는 근무 시간 존중과 연결될 수 있다.

## 성공 기준

- 사용자는 서두르지 않고 메시지를 제어할 수 있다고 느낀다.
